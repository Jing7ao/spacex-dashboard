# 预测层方法论

> 版本：v0.3（v3.3配套）  
> 日期：2026-06-08  
> 定位：供应链映射方法论 v3.3 的预测层配套。**v0.3 关键修正：评分不预测收益（五轮诊断证实）。预测层重心从"评分信号"转向"生态方向+量价时机+排雷"**  
> 状态：**研究稿。v0.3 新增：生态选择溢价量化 + 止损执行时机矩阵 + 买入前检查清单 + 信号平滑**

---

## 一、为什么需要单独的预测层

供应链映射方法论 v3.2 是诊断工具——它验证供应链声称是否真实，但不回答时机问题。实际决策中暴露了三个困境（详见方法论成熟度评估 §九）：

| 困境 | 诊断层能做的 | 诊断层做不到的 |
|------|------|------|
| E1 证实后反而跌 | 验证声称是否真实 | 判断信息是否已被定价 |
| 生态排名第一但已过热 | 统计资金流和涨跌 | 区分趋势和过度拥挤 |
| 虚标清洗可能漏真东西 | 标注"年报未提" | 判断是"不存在"还是"不能说" |

预测层需要建立在诊断层之上，用已有数据（K 线/资金流/评分/事件），输出**可操作的时机信号**。

---

## 二、核心设计原则

### 2.1 不预测价格，预测"条件"

不试图回答"明天涨多少"。回答的是有或无的条件判断：

| 条件类型 | 例子 |
|------|------|
| 趋势确认 | "20 日均线向上 + 连续 3 日收于 MA20 之上" |
| 动能衰竭 | "RSI > 75 且量缩" |
| 资金背离 | "主力连续 3 日流出但价格横盘" |
| 事件触发 | "供应链事件 d5 超额 > 3%" |
| 拥挤警告 | "生态内 60% 标的 RSI > 70" |

### 2.2 用诊断层数据作为先验

预测信号的可信度 = 诊断层的证据等级 × 技术面信号强度。

- E1 标的 + 买入信号 → 高置信度
- 虚标 + 买入信号 → 怀疑（可能是概念炒作）
- E1 标的 + 卖出信号 → 技术面可能反应过度，可能是加仓机会

### 2.3 所有规则必须可回测

预测层的每一条规则必须能用历史 K 线数据验证。不接受"经验感觉"——必须算得出胜率。

## 三、预测层需要回答的三个核心问题

### Q1：生态轮动信号何时有效？

**现状**：生态评分面板每天更新排名，前两名超配。

**问题**：排名变化可能是噪音。一天前 AI 排第一、今天机器人排第一——该换吗？

**研究方向**：
- 排名持续性：连续 N 天保持前 2 的生态，后续表现如何？
- 排名跃迁：从后 3 跳入前 2 的生态，是反转信号还是假突破？
- 生态拥挤度：前 2 生态内标的的 RSI 均值——如果全部 >70，可能过热

**候选规则（待回测）**：
```
生态确认 = 同一生态连续 ≥3 天排名前2 且 生态内RSI均值<65
生态轮动触发 = 生态从后3跃入前2 且 当日资金净流入>0
生态过热警告 = 生态排名前2 且 生态内>60%标的RSI>70
```

### Q2：个股买入信号何时可信？

**现状**：七维评分系统评分 >60 为"偏买入"。

**问题**：评分是静态快照。今天 66 分，明天可能 41 分——信号不稳定。

**研究方向**：
- 评分趋势：评分连续上升 vs 单日跳升，哪种更可靠？
- 评分 + E1 组合：E1 标的的买入信号和虚标标的的买入信号，后续收益差异？
- 资金流确认：评分 >60 且主力净流入 >500 万 vs 评分 >60 但资金流出？

**候选规则（待回测）**：
```
高确信买入 = 评分≥65 + E1/E2标的 + 主力净流入 + 评分连续≥3日上升
低确信买入 = 评分≥65 + 虚标 + 资金流入（可能是概念炒作）
诱多过滤 = 评分≥65 + 资金流出 + 量比<1.5 → 降一级
```

### Q3：什么时候退出？

**现状**：止损 -8%、止盈 +15%。

**问题**：硬止损保护本金但可能频繁触发。止盈可能过早离场。

**研究方向**：
- 止损是否需要区分标的类型？E1 标的的波动率通常低于虚标，止损线是否应该不同？
- 跟踪止损：盈利 >10% 后，止损线从 -8% 上移至成本价或 -3%
- 时间止损：持仓 N 天后如果仍在亏损，即使未到止损线也减仓

**候选规则（待回测）**：
```
E1标的止损 = -8%（正常）
虚标/低确信止损 = -5%（更快止损）
跟踪止损 = 盈利>10%→止损线移至成本价；盈利>20%→止损线移至+5%
时间止损 = 持仓15个交易日仍亏损→减半仓
```

### Q4：当前是什么市场状态？

**为什么重要**：同一条规则在趋势市和震荡市的表现完全不同。趋势市中"RSI>70 超买"可能只是半路，震荡市中"RSI>70"几乎就是顶。

**三态分类**（基于沪深 300，可日频判断）：

| 状态 | 识别条件 | 对所有信号的影响 |
|------|------|------|
| 📈 **趋势市** | 沪深300 的 MA20 > MA60 且 价格在 MA20 之上 | 趋势信号加权（+20%），反转信号降权（-30%） |
| 📊 **震荡市** | 价格在 MA20 和 MA60 之间反复穿越（20日内穿越≥3次） | 反转信号加权（+20%），趋势信号降权（-20%） |
| 📉 **恐慌市** | 沪深300 单日跌幅 >3% 或 VIX 替代指标（20日波动率 >40%） | 所有买入信号暂停。仅执行止损。等待恐慌结束 |

**状态切换规则**：
```
进入恐慌市 → 当日立即暂停所有买入 → 至少观察3个交易日 → 连续2日无>2%跌幅 → 恢复
趋势市→震荡市 → 降低持仓目标（从80%→60%）
震荡市→趋势市 → 提高持仓目标（从60%→80%）
```

**当前局限**：没有 VIX 数据。用沪深 300 的 20 日波动率作为替代——超过 40% 即视为恐慌。

### Q5：多个信号矛盾时怎么处理？

预测层可能同时输出矛盾的信号。需要有明确的优先级：

**信号优先级（从高到低）**：

| 优先级 | 信号类型 | 处理方式 |
|:--:|------|------|
| P0 | 🛑 止损触发 | **无条件执行**。不管其他信号说什么 |
| P1 | 📉 市场恐慌状态 | 暂停所有买入。仅执行 P0 |
| P2 | 🔥 生态轮动变化 | 调仓。但单次调仓不超过总仓位的 30% |
| P3 | 📈 高确信买入 | 在 P2 确定的生态内选标的。单票不超过 20% |
| P4 | 📊 低确信买入 | 仓位减半（相对于 P3）。仅在 P2 生态内 |
| P5 | 🟠 诱多警告 | 不买入。已有持仓的保持观察 |

**矛盾示例**：
- 生态面板说"AI 算力排第一 🔥超配" + 个股信号说"华丰科技🟠诱多" → **信诱多警告**。换 AI 生态内其他标的。
- 生态面板说"低空经济排最后 ❄️回避" + 莱斯信息 RSI<25 超卖 → **信生态排名**。不在回避生态内抄底。

## 三-B、v0.3 关键修正：预测层重心转移

> 2026-06-08 五轮诊断测试后，预测层方法论需要一次根本性修正。

### 不再追求的事

| 放弃 | 原因 |
|------|------|
| 用评分预测个股涨跌 | 打乱标签 88% > 真实 → 评分=噪音 |
| 用 E1 标签选牛股 | E1 vs E3 收益差 +0.89%, t=+0.41 不显著 |
| 用 OLS 学最优权重 | 样本外 r=-0.58 反向预测 |
| 微调因子权重 | 生态选择溢价 87.7% > 任何因子调参 |

### 重心转移到

| 聚焦 | 依据 |
|------|------|
| **生态方向判断** | 光通信 +60% vs 低空 -27%，差异 87.7% |
| **排雷（避开虚标）** | 菲利华 -13.1% 教训 → 买入前检查清单 |
| **执行时机** | 放量跌→立刻执行 vs 缩量跌→等15分钟 |
| **资金二次确认** | 量价+资金持续性→区分机构建仓 vs 游资出货 |

### 预测层 Q1-Q3 的修正

- **Q1（生态轮动）**：不再看评分排名。用 60日/20日真实收益 + 资金持续流入 + 拥挤度三项加权
- **Q2（买入信号）**：不再看评分高低。看买入前检查清单（评分≥40 + 非虚标 + 非回避）+ E1 确认
- **Q3（退出信号）**：不再简单阈值。看止损执行时机矩阵（风控文档 §十）

## 五、学术基础（待深入）

预测层的设计需要以下文献支撑，避免拍脑袋：

### 5.1 趋势跟踪
- Moskowitz, Ooi & Pedersen (2012), "Time Series Momentum", *JFE*  
  核心发现：过去 12 个月收益对未来 1 个月收益有正向预测力。月频趋势在多种资产上有效。
  → 应用：生态排名的动量效应可能比日频评分更可靠。

### 5.2 截面动量（行业轮动）
- Jegadeesh & Titman (1993), "Returns to Buying Winners and Selling Losers", *JF*  
  经典发现：过去 3-12 个月的赢家继续赢，但 1 个月内可能反转。
  → 应用：生态行业轮动应该看中期（20-60 日）而非短期（5 日）。

### 5.3 拥挤交易与反转
- Lou & Polk (2022), "Comomentum", *JFE*  
  发现：当很多股票同时表现出高动量时（"拥挤动量"），后续反转的概率更高。
  → 应用：生态内 >60% 标的 RSI>70 时，生态动量可能不可持续。

### 5.4 信息扩散与价格发现
- Hong & Stein (1999), "A Unified Theory of Underreaction, Momentum, and Overreaction", *JF*  
  核心机制：信息在投资者之间缓慢扩散 → 先产生动量 → 然后过度反应 → 最终反转。
  → 应用：E1 证实可能处于"信息扩散完成"阶段——动量已耗尽，反转风险上升。

### 5.5 凯利与序贯下注
- Thorp (1975), "Portfolio Choice and the Kelly Criterion"  
  已有应用（半凯利）。但序贯下注（连续多次独立押注）的凯利应用需要调整。
  → 应用：同一生态内连续亏损后应该缩仓——即使下一次信号的"凯利比例"算出来很高。

## 六、验证路线图

预测层不能上线就信。必须走验证流程：

| 阶段 | 内容 | 数据需求 | 状态 |
|:--:|------|------|:--:|
| 1 | 生态排名持续性分析 | 8 生态 × 20 日历史排名 | 🔴 待做 |
| 2 | 评分趋势 vs 后续收益 | 50 只标的 × 60 日评分历史 | 🔴 待做 |
| 3 | E1/虚标分组回测 | 已审计标的 + E1/虚标标签 | 🔴 待做 |
| 4 | 资金流确认有效性 | 主力资金流 × 后续收益 | 🔴 待做 |
| 5 | 止损/止盈策略对比 | 不同参数组合 × 历史回测 | 🔴 待做 |
| 6 | 综合规则回测 | 整合全部规则，样本外测试 | 🔴 待做 |

**验证标准**：
- 每条规则必须：胜率 > 50% 且 盈亏比 > 1.5
- 综合策略必须：夏普 > 0.8 且 最大回撤 < 25%
- 未通过验证的规则：不启用，或仅作为辅助参考

## 七、与诊断层的接口设计

预测层不替代诊断层——它叠加在诊断层之上，用诊断层的输出作为先验：

```
诊断层输出 → 预测层增强
─────────────────────────
E1 标的   → 买入信号 × 1.2（信任加权）
虚标      → 买入信号 × 0.6（怀疑折扣）
生态前2   → 只在前2生态内考虑买入
生态后2   → 只考虑卖出/减仓，不买入
事件E1    → d5超额>3% → 短期动量确认
```

**关键原则**：诊断层过滤掉"不该买的"（虚标/伪供应链），预测层决定"什么时候买该买的"。

## 九、无回测条件下的验证路径（v0.2 新增）

> §2.3 要求"所有规则必须可回测"，但当前没有历史交易记录。本节提供替代方案。

### 9.1 前向验证（Forward-Testing）

不依赖历史数据。从现在开始记录，一个月后回头看：

**方法**：预测层每天输出信号 → 记录 → 但不执行（或仅小仓位验证）→ 30 天后统计：
- 生态排名前 2 的生态实际涨跌幅 vs 后 2 的生态
- 高确信买入信号出现后 5 日/20 日的平均收益
- 诱多信号出现后 5 日是否确实下跌

**工具**：predict.html 的信号可以自动记录到 localStorage，30 天后导出分析。

### 9.2 逻辑一致性检验（不需要数据）

某些规则的正确性可以从逻辑推导，不需要回测：

| 规则 | 逻辑检验 |
|------|------|
| 虚标止损更紧（-5% vs -8%） | ✅ 逻辑成立。虚标的基本面支撑更弱，下跌时无"价值锚" |
| E1 标的在超卖时加仓 | ⚠️ 方向对但幅度未知。E1 有基本面锚，但不知道"超卖到多少是底" |
| 生态拥挤度 >70% RSI 时警告 | ✅ 逻辑成立。拥挤 → 后续买家减少 → 动量衰竭 |
| 市场恐慌时暂停所有买入 | ✅ 逻辑成立。恐慌时所有相关性趋向 1，分散化失效 |

### 9.3 小仓位实盘验证（最可靠）

对于无法通过逻辑检验的规则，用真实交易验证——但仓位小到不影响整体：

- 每条待验证规则的验证仓位：总资金的 **2-3%**（15 万 × 3% = 4500 元）
- 验证周期：至少 5 笔交易
- 判定标准：5 笔中 ≥3 笔盈利 → 规则暂定有效；≤2 笔 → 废弃或修改参数

## 十一、预测规则手册（v0.2 新增）

> 把所有规则整理成一张决策表。每天开盘前/盘中对照执行。

### 第一步：判断市场状态（Q4）

```
沪深300 MA20 > MA60 且 价格 > MA20  →  📈 趋势市
价格在 MA20/MA60 间穿越 ≥3次/20日  →  📊 震荡市
单日跌幅 >3% 或 20日波动率 >40%    →  📉 恐慌市
```

### 第二步：选择生态（Q1）

```
趋势市：
  生态排名前 2 且 持续性 ≥3 天 且 RSI均值 <70  →  超配（各 30-40%）
  生态排名前 2 但 持续性 <3 天                  →  标配（各 15-20%）
  生态排名后 2                                   →  不买入，已有持仓考虑减仓

震荡市：
  生态排名前 2 且 RSI均值 <50                    →  超配（反转机会）
  生态排名前 2 但 RSI均值 >65                    →  标配（可能过热）
  生态排名后 2 且 RSI均值 <35                    →  关注（超卖反弹机会）

恐慌市：
  不买入任何标的。仅执行止损。
```

### 第三步：选标的（Q2）

```
在选定生态内：
  E1/E2 + 评分≥70 + 主力流入 + 评分连续3日上升  →  🟢 高确信（可用正常仓位）
  E1/E2 + 评分≥65 + 主力流入                      →  🟢 高确信（可用正常仓位）
  评分≥65 + 主力流入 + 虚标                       →  🟡 中确信（仓位减半）
  评分≥65 + 资金流出                              →  🟠 诱多（不买）
  评分<60                                          →  不考虑（除非RSI<25超卖+E1）
```

### 第四步：定仓位（Q3+Kelly）

```
基础仓位 = 凯利半仓（默认 15%）
× 市场状态系数（趋势市 1.0, 震荡市 0.75, 恐慌市 0）
× 信号确信度系数（高确信 1.0, 中确信 0.5, 低确信 0）
× 生态排名系数（前2 1.0, 中2 0.7, 后2 0）
= 最终仓位

止损：
  E1/E2 标的 → -8%
  虚标/低确信 → -5%
  盈利 >10% → 止损线移至成本价
  盈利 >20% → 止损线移至 +5%
  持仓 15 日仍亏损 → 减半仓（不管是否到止损线）
```

### 第五步：信号冲突时（Q5）

```
P0 止损触发           → 无脑执行
P1 恐慌市             → 停止一切买入
P2 生态排名变化       → 调仓（单次 ≤30%）
P3 高确信买入         → 在 P2 生态内执行
P4 低确信买入         → 仓位减半
P5 诱多               → 不买
```
